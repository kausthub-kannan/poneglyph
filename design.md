# Poneglyph — Research-Oriented Obsidian Plugin

## Overview

Poneglyph is an Obsidian plugin designed exclusively for research vaults.

It turns every markdown node in the Obsidian graph into a deeply researched,

cited artifact — generated by a deep research agent (deepagent). The graph

itself becomes a living, navigable map of your research understanding.

---

## Node Creation

Each node in the graph is a markdown file. Nodes can be created in two ways:

### 1. Manual Creation (User Query)

- The user selects one or more existing nodes in the graph as parent context

- The user provides a research query

- The deepagent uses the selected nodes' content + the query as input

- It performs deep research and generates the markdown file with cited sources

### 2. Auto Generation (Recursive)

- The user selects existing nodes and triggers auto-expand

- An LLM reads the selected nodes' content and auto-generates a well-formed

  research query

- The deepagent performs research and creates the new node

- This continues recursively from newly created nodes

- A hard node depth limit (user-configured) stops runaway generation

- Auto-generated nodes are always created with status: draft and require

  human ratification before they can serve as parent context for further

  auto-generation

---

## Node Metadata (Frontmatter)

Every node carries the following frontmatter:

```yaml

---

title: ""

status: draft              # Tag Set A: draft | researched | verified

relation_to_parent:        # Tag Set B: supports | contradicts | elaborates

role: foundational         # Tag Set D: foundational | tangential

confidence: 0.00           # Float, agent-assigned, 0.0 to 1.0

created: YYYY-MM-DD

query: ""                  # The query used to generate this node

parent_nodes:

  - ""

sources:

  - ""                     # Source keys referencing SOURCES.md

---

```

### Tag Set Rules

**Tag Set A — Lifecycle Status**

- `draft` — node created, content not yet verified

- `researched` — deepagent has completed research and generated content (auto-set)

- `verified` — human has reviewed and approved the node (manual only)

- Promotion from `researched` → `verified` is always a human action, never automatic

**Tag Set B — Node Role**

- `foundational` — a seed node, manually created by the user as a base

- `tangential` — an auto-generated node on the periphery of the core research

## Vault Structure

This plugin assumes the Obsidian vault is used exclusively for research.

### SOURCES.md

A single vault-wide citation ledger maintained automatically.

- Every time a node is created, its cited sources are added/updated here

- Sources are deduplicated by a unique source key (e.g. smith2021)

- Format: a table sorted by h-index (descending)

| Key | Title | Authors | Year | Journal | H-Index | DOI / Link |

|-----|-------|---------|------|---------|---------|------------|

## Graph Visualization

Obsidian's native graph coloring (via CSS snippets or Style Settings plugin)

can reflect node metadata visually:

- Status A maps to node color: grey (draft), blue (researched), green (verified)

- Relation B maps to edge color: green (supports), red (contradicts), yellow (elaborates)

- Role D maps to node size or shape: larger (foundational), smaller (tangential)

---

## Key Constraints

- Auto-generated nodes in `draft` status cannot be parent context for further

  auto-generation until ratified by the user

- The vault contamination guard warns if non-research notes are detected and

  pulled into agent context

- SOURCES.md is never used as a parent node for research context

- The hard depth limit for recursive auto-generation is user-configurable

  (recommended default: 3)
